## Problem Statement

Listojo is a classifieds/rental marketplace targeting DFW renters and roommate-seekers. The codebase is ~80% launch-ready with a smart matching engine (FMM), guest chat, inquiry system, Google Maps integration, city waitlist, and admin portal. The gap is zero supply: no landlord listings, no tenant traffic, no demand validation.

## YC Diagnostic Summary

**Demand evidence:** None yet. Pre-product.

**Status quo being displaced:** Zillow/Apartments.com (expensive/gated for small landlords). Also Craigslist and Facebook Marketplace (free but no trust, no filtering, spam).

**Target user:** Small DFW landlords (1-5 self-managed units) who find Zillow too expensive and Craigslist/Facebook too manual/untrusted. Renters are the demand side, but supply (landlord listings) is the blocker.

**Wedge market:** DFW. Confirmed as strong (7.8M people, high rental demand, large independent landlord population). Geographic focus on outreach, not on product gating.

**Monetization:** Free for now. Monetize later via premium features, screening, or leads. Classic marketplace playbook.

## Key Insight (Eureka)

The FMM smart matching engine is the codebase's centerpiece (150+ lines in `listings/views.py`), but it only becomes valuable at 100+ listings. The real blocker is getting the first 50 DFW listings — not building better search. **The MVP is a zero-friction free listing tool, not a smart search engine.** Smart matching is the long-term moat; free supply is the launch wedge.

## Agreed Premises

1. Supply is the real constraint — smart matching doesn't matter at <50 listings.
2. DFW is the right wedge market. Outreach focus there; product can be city-agnostic.
3. Free for landlords is the wedge against Zillow. Monetization comes later.

## Chosen Approach: A+C — Ship What You Have + Community Seeding

**Summary:** Launch the current build. Seed supply by manually reaching out to DFW landlords (Facebook groups, Reddit r/Dallas, NextDoor, direct outreach). Learn what breaks before building more features.

### Why this beats the alternatives

- The code is 80% done. Another sprint of features before launching is the enemy.
- Community seeding (C) is zero-code, fastest path to real signal.
- Learning from 5 real landlords posting will do more for the product than any planned feature.

### What "launch" means here

1. Flip `LAUNCH_ACTIVE = False` in `settings.py` to open the platform to all cities.
2. Manually seed 10-20 DFW listings (post as the landlord, or reach out to real ones).
3. Post in 3 DFW Facebook rental groups, r/Dallas, r/DFW, NextDoor.
4. Drive 5 landlords to the create listing flow. Watch them. Fix what breaks.
5. Ship after you've seen 5 people post without help.

## What's Missing Before Launch

| Gap | Severity | Fix |
|-----|----------|-----|
| Zero tests | High | Add tests for FMM scoring, listing create, chat. ~15min with CC. |
| SQLite in production | High | Switch to PostgreSQL before any real traffic |
| SECRET_KEY default is insecure | High | Ensure `.env` is set before deploy — default is `dev-only-secret-key-change-me` |
| No supply plan | Critical | Community seeding (Approach C) |
| STATIC_ROOT not set | Med | Add `STATIC_ROOT = BASE_DIR / 'staticfiles'` to settings.py |
| No CI/CD | Med | GitHub Actions for test + deploy |
| Media files committed to git | Low | Add `media/` to .gitignore, use S3 or similar |
| `tags` stored as comma-separated string | Low | Tech debt. Works for MVP. M2M migration later. |

## 12-Month Vision

```
  CURRENT STATE                  LAUNCH (1-3 mo)           12-MONTH IDEAL
  Zero listings               20-50 DFW listings        500+ listings, 3 metros
  No demand validation        First real inquiries       Recurring users, word of mouth
  Smart matching unused       FMM starts generating      FMM is primary search UX
  SQLite                      PostgreSQL                 PostgreSQL + caching
  No tests                    Core tests added           Test suite CI-gated
  Admin portal only           Landlord self-serve good   Landlord dashboard + analytics
```

## NOT in Scope (for launch)

- AI-generated listing descriptions
- Background checks / tenant screening
- Payment / deposit handling
- Mobile app
- Real-time notifications (WebSocket)
- Map clustering / advanced map UX

## Deferred (Post-Validation)

- Monetization model (premium placement, lead gen, screening)
- Houston / Austin expansion (configs already stubbed in settings.py)
- Multi-image gallery reordering UX
- Email digest for new listings matching saved searches
